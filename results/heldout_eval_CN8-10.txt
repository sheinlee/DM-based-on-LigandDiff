Held-out generative evaluation of the single-ligand DM (DM-based-on-LigandDiff) models.

Protocol: 40 held-out Ln seeds, CN 8-10 filter, save-all generation (generate_saveall, n_samples=8),
xtb GFN-FF relaxation, then per-ligand Validity / Uniqueness / Novelty + complex-level Yield +
whole-complex topology Recall. SAME held-out reference (Ln_data_new/val), SAME novelty reference
(ref_Ln: 1478 SMILES / 5624 WL graphs), SAME metric code as the coordination-site-conditioned Multi
model -> the two blocks below and the reference rows are directly comparable.

Our strict metric: RDKit rdDetermineBonds (charge 0,+-1,+-2) + SanitizeMol per ligand, on relaxed
geometry. (Under the literature's own OpenBabel-on-raw algorithm the same class of model scores
~94% validity; see the companion Multi-Liganddiff-on-Ln-complexes repo for that analysis.)

READING
- Bigger Ln data alone does NOT lift the plain model: `full` (larger CN 4-12 Ln_data_new, ~ep276)
  lands at baseline level (Validity 59.8 vs 60.4), well below the coordination-site-conditioned
  Multi (68.0). More data + longer training did not close the gap.
- Staged TM->Ln transfer (ep134) has the LOWEST validity (54.0) and lowest target fidelity
  (partial 0.473): more diverse/novel but less faithful, and undertrained at ep134 (a confound).
- Conditioning > data / capacity / transfer: telling the model WHERE the coordination sites are
  (Multi) beats bigger data, more parameters (B), and TM pre-training.
- All models score 0% exact topology recall -- consistent with the entire de novo
  coordination-complex generation field.
